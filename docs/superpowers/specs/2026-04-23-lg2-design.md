# LG2 — Late Goal 2e mi-temps (design)

- **Date** : 2026-04-23
- **Auteur** : Benjamin (brainstorming avec Claude)
- **Statut** : design validé, prêt pour plan d'implémentation

## 1. Objectif

Ajouter une nouvelle stratégie de trading sportif football, LG2 (Late Goal 2e mi-temps), qui identifie les matchs susceptibles de produire un but après la 80e minute (temps additionnels inclus). La stratégie coexiste avec FHG et DC dans l'application, sans impact sur les algos existants.

## 2. Définition du signal

Pour chaque équipe (séparément côté domicile et côté extérieur), on calcule un **streak consécutif** de matchs récents dans lesquels au moins un but a été marqué après la 80e minute, peu importe par qui.

- `LG2_MIN_MINUTE = 80` — un but en `min >= 80` compte (inclut `90+n`).
- `LG2_STREAK_MIN_MATCHES = 3` — il faut au moins 3 matchs d'historique pour évaluer.
- `LG2_STREAK_MOYEN = 3` — streak 3 consécutifs → `moyen`.
- `LG2_STREAK_FORT = 4` — streak ≥4 consécutifs → `fort`.

Un match d'historique est comptabilisé avec le rôle domicile/extérieur correspondant au match à venir : si l'équipe X joue à domicile dans le match analysé, on ne considère que ses précédents matchs à domicile. Même logique côté extérieur.

### Scénarios

| signal_type | Déclencheur | Confidence |
|-------------|-------------|-----------|
| `LG2_A` | streak équipe DOM ≥ 3 | `moyen` si streak=3, `fort` si streak≥4 |
| `LG2_B` | streak équipe EXT ≥ 3 | `moyen` si streak=3, `fort` si streak≥4 |
| `LG2_A+B` | les deux côtés actifs simultanément | `fort_double` |

Pas de scénario C/D, pas de confirmation indépendante, pas de veto H2H.

## 4. Algorithme (pseudo-code)

```js
const LG2_MIN_MINUTE = 80;
const LG2_STREAK_MIN_MATCHES = 3;
const LG2_STREAK_MOYEN = 3;
const LG2_STREAK_FORT = 4;

function matchHasGoalAfter80(match) {
  const events = Array.isArray(match.goal_events) ? match.goal_events : [];
  return events.some(e => e.min >= LG2_MIN_MINUTE);
}

function computeLG2Streak(matches) {
  // matches doit être trié DESC par match_date
  let streak = 0;
  for (const m of matches) {
    if (matchHasGoalAfter80(m)) streak++;
    else break;
  }
  return streak;
}

function analyzeLG2(homeMatches, awayMatches) {
  if (homeMatches.length < LG2_STREAK_MIN_MATCHES &&
      awayMatches.length < LG2_STREAK_MIN_MATCHES) {
    return { isAlert: false, reason: 'insufficient_history' };
  }

  const streakHome = computeLG2Streak(homeMatches);
  const streakAway = computeLG2Streak(awayMatches);

  const aActive = streakHome >= LG2_STREAK_MOYEN;
  const bActive = streakAway >= LG2_STREAK_MOYEN;

  if (aActive && bActive) {
    return {
      isAlert: true,
      signalType: 'LG2_A+B',
      confidence: 'fort_double',
      factors: { streakHome, streakAway },
    };
  }
  if (aActive) {
    return {
      isAlert: true,
      signalType: 'LG2_A',
      confidence: streakHome >= LG2_STREAK_FORT ? 'fort' : 'moyen',
      factors: { streakHome, streakAway },
    };
  }
  if (bActive) {
    return {
      isAlert: true,
      signalType: 'LG2_B',
      confidence: streakAway >= LG2_STREAK_FORT ? 'fort' : 'moyen',
      factors: { streakHome, streakAway },
    };
  }
  return { isAlert: false, factors: { streakHome, streakAway } };
}
```

## 5. Persistance (Supabase)

Table `alerts` inchangée. Les alertes LG2 réutilisent les colonnes existantes :

| Colonne | Valeur LG2 |
|---------|-----------|
| `signal_type` | `LG2_A`, `LG2_B`, `LG2_A+B` |
| `confidence` | `moyen`, `fort`, `fort_double` |
| `algo_version` | `lg2_v1` |
| `fhg_factors` | jsonb `{ streakHome, streakAway }` (réutilisation, cf. note ci-dessous) |
| `fhg_pct`, `fhg_confidence`, `dc_*` | `NULL` |

Pas de migration SQL : les colonnes FHG réutilisées sont déjà nullables.

> **Note** : on réutilise `fhg_factors` pour éviter une migration. Le front-end discrimine via `algo_version`. Si la confusion sémantique devient gênante, on renommera en `factors` (migration simple) dans une itération future.

## 6. Génération (`generate-alerts.js`)

Pour chaque match d'une ligue active, après l'évaluation FHG/DC existante, on exécute LG2 :

1. Extraire `homeMatches` depuis `home_team.lastx` déjà chargé, filtré sur les matchs où l'équipe joue à domicile.
2. Idem pour `awayMatches` côté extérieur.
3. `analyzeLG2(homeMatches, awayMatches)` → si `isAlert`, insert dans `alerts` avec `algo_version='lg2_v1'`.
4. Insert ignore-duplicates sur `(match_id, signal_type)`.

**Aucun appel API supplémentaire** — les `lastx` et `H2H` sont déjà chargés pour FHG. Pas de nouveau cron.

## 7. Vérification des résultats (`check-results.js`)

On étend le switch existant :

```js
const LG2_TYPES = ['LG2_A', 'LG2_B', 'LG2_A+B'];

if (FHG_TYPES.includes(signalType)) {
  newStatus = evaluateFHG(matchData);
} else if (LG2_TYPES.includes(signalType)) {
  newStatus = evaluateLG2(matchData);
} else if (signalType === 'DC') {
  newStatus = evaluateDC(matchData, alert.dc_best_side);
}

function evaluateLG2(matchData) {
  const mins = parseGoalMinutes(matchData);
  if (mins.length === 0) {
    console.warn(`[check-results] LG2 ${matchData.id}: no timings, marking lost (conservative)`);
    return 'lost';
  }
  return mins.some(m => m >= 80) ? 'validated' : 'lost';
}
```

Staleness 48h et évaluation conservatrice réutilisés sans changement.

## 11. Tests

### `lg2.cjs` et `lg2.js` (~20 tests chacun)

- `matchHasGoalAfter80` : `min=79` (false), `min=80` (true), `min=90` (true), `min=90+3` (true si parsé), événements non-tableau, tableau vide.
- `computeLG2Streak` : 3 consécutifs, streak brisé après 2, 4+ consécutifs, tableau vide, tri respecté.
- `analyzeLG2` : scénario A seul (streak home 3, away 1), B seul, A+B, A fort (streak≥4), A moyen (streak=3), pas d'alerte, historique insuffisant côté home ET away.

### `check-results.js`

- `evaluateLG2` : goals empty → lost, goals <80 → lost, goal=80 → validated, goal=90 → validated, goal en temps add. parsé à 90 → validated.
